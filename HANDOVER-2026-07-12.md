## P1. New bug: `sch_get_connectivity` silently drops off-page components

The highest-value fix. Same silent-data-loss class as the six bugs in the
README table.

- `src/extension/handlers/sch-document.ts:44` calls
  `eda.sch_PrimitiveComponent.getAll('part', true)`. The
  `allSchematicPages=true` flag is the exact one documented as ignored by
  EDA Pro in `sch-component.ts:185-217` (bug 2): it returns only the
  active page.
- Mechanism: the netlist itself IS project-wide (fetched via
  `getNetlistFile`), but `uniqueToPrimitive` is built only from the
  active-page components (`sch-document.ts:47-56`), and every downstream
  loop gates on it: `sch-document.ts:67` (`if (!uniqueToPrimitive[uniqueId])
  continue;` when building BFS indices) and again at ~:127 when building
  the nets/components views. Result: on a multi-page project, components
  and their pins on other pages are silently absent from connectivity
  output, and depth-BFS expansion cannot traverse through them.

**Proposed fix:** reuse the verified per-page fan-out from
`sch-component.ts:169-241` (page restore in `finally`, landing-page
verification) to build the full component map; or, after the P3 types
upgrade, prefer the new `SCH_Net` API (see P3) which is project-wide by
design. Either way add a regression note: `sch-document.ts` and
`sch-component.ts` must not disagree about `allSchematicPages` again.

Caution: the fan-out switches pages, so this collides with H11 (no request
serialisation, below). Doing H11 first, or at least in the same session, is
strongly advised.

## P2. Defensive handling for fresh upstream runtime bugs

Filed against `easyeda/pro-api-sdk` in July 2026
(https://github.com/easyeda/pro-api-sdk/issues). These are EDA-side bugs;
the MCP should guard or document them because callers here are LLMs that
trust tool output.

1. **Issue #27 — `sch_Drc.check(..., includeVerboseError=true)` returns a
   runtime boolean, not the typed `Promise<Array>`.**
   `src/extension/handlers/sch-document.ts:16-17` passes `true` and the
   comment at :12-15 assumes an array comes back. Verify live; if a boolean
   arrives, wrap: return a structured `{passed: boolean, errors?: []}` and
   say so in the `sch_drc_check` tool description
   (`src/bridge-daemon/tools/sch-read-tools.ts:236`).
2. **Issue #34 — pour reflow uses a rules snapshot from document open.**
   Rules written via API after the PCB is open read back correctly but do
   not affect reflow until `closeDocument` + `openDocument`. Proposed:
   after `pcb.drc.saveRuleConfig`-class writes, either auto reopen the
   document or append a warning to the rule-write and pour tool
   descriptions telling the model to reopen before rebuilding pours.
3. **Issue #33 — pad `number` reads back null until document reload** after
   API component placement, and DRC "Netlist Error" gives no structured
   detail. Affects any workflow that places a component then immediately
   reads its pads. Proposed: document in the placement tools, or reopen the
   document after placement when pads will be read.
4. **Issue #32 — internal plane (PLANE) anti-pads are not cut for
   different-net vias created after plane generation**; rebuilding the pour
   does not fix it, DRC reports Plane Zone to Via. Proposed: warning in via
   creation and pour rebuild tool descriptions; this is a fab-risk footgun.

## P3. Upgrade `@jlceda/pro-api-types` 0.2.9 → 0.3.5 and adopt new APIs

`package.json` pins `^0.2.9`; npm is at 0.3.5. Diff results (verified by
extracting both `.d.ts` files):

- **Safe to upgrade:** the only removed method (`SYS_Unit.getSystemDataUnit`)
  and newly deprecated ones (four `LIB_Classification` methods) are not
  referenced anywhere in `src/`. Note the 0.3.5 `index.d.ts` has a
  different file layout (React imports at top, global declare block), so
  re-run `npm run typecheck` on both configs and expect casts like the
  `(eda.sch_PrimitiveComponent as any)` at `sch-document.ts:44` to keep
  compiling either way.
- **New APIs worth adopting:**
  - `SCH_Net.getAllNets() / getNet(net) / getAllNetsName() /
    getCurrentProjectAllNets()` (all marked `@alpha`). A direct pin-to-net
    read. `getCurrentProjectAllNets` returns per-schematic net info across
    the whole project — a candidate to replace or back up both the bug 3
    netlist-file workaround (`sch-netlist-utils.ts`) and the P1 fix.
    Measure latency and shape on a real project before switching; keep
    `getNetlistFile` as fallback since these are alpha.
  - `PCB_Drc.startRealTimeDrc / getRealTimeDrcStatus / stopRealTimeDrc`:
    new tool candidates alongside `pcb.drc.check`.
  - `PCB_ManufactureData.getIpc2581CFile` and
    `getAutoRouteJsonFileForJRouter`: new export formats for
    `manufacture-tools.ts`.
  - `SCH_ManufactureData.getBomTemplates / getBomTemplateFile /
    uploadBomTemplateFile / deleteBomTemplate`: BOM template management,
    pairs with the metadata-preservation work.
  - `SYS_Math` (union, intersection, contains, bbox, centroid, etc.):
    could serve pour/region tooling and replace hand-rolled geometry.
  - `DMT_Event.addEditorTabEventListener`: subscribe to tab switches to
    keep `list_instances` fresh (fixes the stale `pushInstanceInfo`
    nice-to-have in AUDIT.md).
  - Also new: `LIB_SimulationModel`, `PCB_RayTracerEngine`,
    `SCH_PrimitiveObject` / `ISCH_PrimitiveObject`.

## Gotchas (carried over, still true)

- Repo conventions: tabs; docs in British English, no em dashes.
- Bridge-daemon tests spawn real daemons and take 60-90 s; in a sandboxed
  shell with a ~45 s cap, run with `--test-name-pattern` in halves.
- The test fixture symbol map only contains `sym-resistor-uuid`, so
  `model.palette.netport` is empty in tests; `addNetportAt` throws there.
- `makeXxxLine()` factories return raw tuples; outside the writer wrap with
  `wrapAsParsedLine()` before pushing into a `ParsedLine[]` stream.
- Same-version `.eext` reinstalls are a no-op; always bump
  `extension.json` version.
- `(eda.sch_PrimitiveComponent as any)` casts exist because 0.2.9 types
  lag the runtime; some may become removable after the 0.3.5 upgrade.
